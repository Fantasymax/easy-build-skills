# Mode B 任务项目化规则（V0.8 — 5 维评分系统）

> **何时读**：用户选 Mode B 后，前置 1 阶段需"项出 5–8 个候选任务让用户选 1 个"——这份文档详细规定 AI 如何从 C2 + A4 + heuristics 项出真正有质量的候选清单。
>
> **核心设计哲学**：用户是小白——他们说不出"我要把哪个任务做成 skill"。AI 必须**主动项出来给他们选**。同时遵守 §B7 选项法 **3+2+1**：3 强相关 + 2 创意发散 + 1 自定义兜底。
>
> **来源**：从 cowork V0.4.1 task-shortlist-rules.md 借用 + V0.8 整合。

---

## Part 1 — 输入数据

AI 项任务时**必须**综合以下来源：

| 来源 | 字段 | 用途 |
|---|---|---|
| 共享核心 C2 | 用户自报的"最近 7 天重复 ≥ 2 次且单次 ≥ 10 分钟"任务清单 | 项任务的**主池**（必须从这里出） |
| 共享核心 C3 | 工具栈 + 数据源 + 模板 | 用来评估"是否可 skill 化" |
| 共享核心 C4a | 满意经历（V0.8 拆分） | 提示"哪类任务用户已经在用 AI" → 优先级 |
| 共享核心 C4b | 失败经历（V0.8 拆分） | 提示"哪类任务用户希望 AI 做但失败过" → 高优先级 |
| Mode A A1 | 触发情绪 | 提示"哪类任务用户最累" → 优先级 |
| Mode A A4 | 最小可执行动作 | 主目标 = 候选 No.1 默认 |
| Mode A A9.5 | 协作深度 | 决定是不是"集中攻坚型"——影响 shortlist 中复合任务的占比 |

**禁忌**：
- ❌ 不许从 C2 之外凭空造任务（用户没说过的不许出现）
- ❌ 不许把"AI 已经能做得很好的通用任务"放进 shortlist（如"翻译"、"压缩"——这些不需要 skill）

## Part 2 — 候选任务的 5 个评分维度

每个 C2 任务按 5 维度打分（满分 50），总分 ≥ 25 才入 shortlist：

| 维度 | 权重 | 评分规则 |
|---|---|---|
| **频次** | ×2 | 每周 ≥ 5 次 → 10；每周 1–4 次 → 6;每月 1 次 → 2 |
| **耗时** | ×2 | 单次 ≥ 1 小时 → 10；30–60 分钟 → 6；< 30 分钟 → 2 |
| **可拆解** | ×1.5 | 步骤可清晰分解 → 10；需用户思考但能描述 → 6；纯灵感型 → 2 |
| **隐性知识密度** | ×1.5 | 有"行内规矩 / 必须用 X 术语 / 我的固定模板" → 10；通用最佳实践就够 → 4 |
| **崩溃修复价值** | ×1 | C4b 提到 AI 在此处常出错 → 10；从未试过 AI → 6；AI 已胜任 → 2 |

**总分阈值**：
- ≥ 35：**强候选**（必入 shortlist）
- 25–34：**弱候选**（按需入）
- < 25：**不入 shortlist**

## Part 3 — Shortlist 组装规则（3+2+1 模式）

### 任务类型标签清单

每个 C2 任务先打类型标签，再做多样性筛选：

| 任务类型 | 关键词触发 | 例子 |
|---|---|---|
| 写作（write）| 写、起草、生成内容、撰写 | 写公众号文章、写代码、写文档、写脚本 |
| 改进（refine）| 改、修订、润色、压缩、优化 | 改稿、refactor 代码、改 PR |
| 决策（decide）| 选、决定、挑、拍板、判断 | 选题、选方案、做产品决策 |
| 整理（organize）| 整理、分类、归档、聚合、做表 | 素材分类、归档资料、做月度报告 |
| 沟通（communicate）| 回、答、私聊、客服、外发 | 回邮件、回评论、客户答疑 |
| 检查（check）| 检查、review、查、verify、扫描 | PR review、AI 痕迹检查、合规审查 |
| 数据/分析（analyze）| 算、统计、追踪、监控、画图 | 数据分析、看报表、跑监控 |
| 创作（create）| 创作、设计、构思、原型 | 创意构思、设计原型、出概念 |
| 学习（learn）| 学、研究、看书、记笔记、整理课件 | 看书做笔记、整理课件、研究论文 |

**类型识别规则**：
- AI 扫描每个 C2 任务的描述
- 匹配上面任一关键词 → 打对应类型标签
- 多关键词命中 → 打多个标签（如"改稿"既是 refine 也是 write）
- 完全不匹配 → 标 `unknown`，由 AI 推断或保留用户原话

### 3 个"强相关"候选（默认排前 3）

按总分降序取前 3 个 C2 任务。**额外约束**：

1. 第 1 名必须是 A4 主目标（如 A4 已经在 C2 中）；A4 不在 C2 中则 A4 为第 1 名
2. **类型多样性约束**：3 个候选**必须涵盖 ≥ 2 个不同类型标签**
   - 算法：从总分降序候选中依次选 → 如新加入候选与已选候选**所有标签都重复**则跳过 → 选下一个
   - 例：小马的 6 个 C2 任务中，"写公众号长文"和"改稿"都是 write+refine 类型，**只能选其一进 shortlist**
3. 第 2、3 名按总分排，但要剔除与第 1 名"过于相似"的（标签全重叠）

### 同类挤压时的处理（针对小马这种"写作密集型"用户）

如果用户 6 个任务中 5 个都是 write 类型 → 强相关 3 名可能凑不出 ≥ 2 类型多样性。
处理：
- 仍取总分前 3 → 即使类型重复
- **但在 shortlist 输出时显式告知用户**："你的高频任务以写作为主——3 个候选都是写作类，是否考虑把其中 2 个合并？"

### 2 个"创意发散"候选

**目的**：让用户跳出"自己说过的"，看到"潜在想要的"。规则：

| 创意发散类型 | 来源 | 例子 |
|---|---|---|
| **跨界组合**：把 C2 任务 A 与 C2 任务 B 组合成新任务 | C2 中相邻 2 项 | "选题 + 改稿 = 一个'本周稿件全流程'工作流" |
| **反向任务**：把用户**没在做但应该做**的事项出来 | C4b 崩溃 + A8 红线 | "写完发布前的 AI 痕迹检查"——如果 C4b 提到怕 AI 痕迹 |
| **隐藏自动化**：用户每次手动做的"碎片小事" | A1 触发场景的"5 分钟前" | "整理今天看过的 5 篇文章中的素材到 Notion" |
| **元任务**：让 AI 帮用户管理 AI 协作本身 | A9.5 + 多次 AI 对话经验 | "审计我本周与 AI 的所有对话，提炼出 3 个新的'我应该让 AI 做'的任务" |

**强约束**：创意发散候选必须**真的出格**（不是另一个普通任务）；否则不如不出。

### 1 个"自定义兜底"

固定文案：

> "🖊 **我已经知道我要抽什么** —— 直接告诉你（请用一句话描述）：__________"

熟练用户专用。

## Part 4 — 项任务呈现给用户的格式

### UI 模式（AskUserQuestion，≤ 4 选项硬限制 → 拆 2 题）

**第 1 题**：3 强相关 + 自定义出口

```python
AskUserQuestion(questions=[{
    "question": "我从你的回答里项了 3 个最强候选任务，挑 1 个我们来把它做成 skill：",
    "header": "强相关任务",
    "multiSelect": False,
    "options": [
        {"label": "改稿润色", "description": "每周 4 次，每次 1–2h，痛点最重"},
        {"label": "选题", "description": "每周 2–3h，需要你的判断"},
        {"label": "素材分类", "description": "每天 30min，可拆解、规则化"}
    ]  # AskUserQuestion 自动加 Other 出口给"我自己说"
}])
```

**第 2 题（如用户选 Other）**：2 创意发散 + 自定义

```python
AskUserQuestion(questions=[{
    "question": "或者，你看看这两个'你可能没想到但其实需要的'候选：",
    "header": "创意发散",
    "multiSelect": False,
    "options": [
        {"label": "AI 痕迹检查器", "description": "发布前过一遍稿件查\"AI 味\""},

}])
```

## Part 5 — 用户反馈处理

### 用户选 ① ② ③（强相关）

直接进入 Mode B 任务问卷（B1–B10），用所选任务作为后续追问的锚点。

### 用户选 ④ ⑤（创意发散）

按 §B7.4 规则**多追问一句**："这是真的想要做，还是觉得有趣？"——区分"反应"与"承诺"。如用户确认要做，进入 Mode B 问卷。

### 用户选 ⑥（自定义）

- AI 解析用户描述
- 检查是否在 C2 任务清单中：
  - 在 → 用户未意识到这是高频任务，AI 提示后进入 Mode B
  - 不在 → 用户提了 C2 之外的新任务，AI 反问："这个任务你最近 7 天做过几次？单次多久？" 补全 5 维评分后再进入

### 追加补充

任意选项后允许追加 ≤ 50 字的"还想说……"——AI 据此微调 Mode B 问卷的具体追问角度。

---

## Part 6 — 与 Mode B 问卷的衔接

任务选定后，**所有 B1–B10 题**都用所选任务作为隐含 context（不是泛泛问"你的工作"）：

- B1 触发场景：选定任务的具体触发瞬间
- B2 输入材料：选定任务的实际输入
- B3 SOP：选定任务的步骤
- B4 判断节点：选定任务里需要决策的点
- B5 行内规矩：选定任务专属的硬规则
- ...
